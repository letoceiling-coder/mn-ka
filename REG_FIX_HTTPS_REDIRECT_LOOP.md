# Исправление бесконечного редиректа на HTTPS

## Проблема
На HTTPS все еще происходит редирект 301: `Location: https://mn-ka.ru/`

## Возможные причины

1. Редирект с `https://mn-ka.ru` на `https://mn-ka.ru/` (трейлинг слеш)
2. Редирект в `.htaccess` в `public/` директории
3. Настройки ISPmanager (чекбокс "Перенаправлять HTTP-запросы в HTTPS")

## Решение

### Шаг 1: Отключите редирект в ISPmanager

В настройках сайта `mn-ka.ru` в ISPmanager:
- **Отключите** чекбокс "Перенаправлять HTTP-запросы в HTTPS" (если он включен)
- Редирект HTTP → HTTPS уже настроен в `.htaccess`, поэтому двойной редирект не нужен

### Шаг 2: Проверьте .htaccess в public/

```bash
cd /var/www/u3384357/data/www/mn-ka.ru

# Проверьте .htaccess в public/
cat public/.htaccess | head -25

# Убедитесь, что там нет дополнительных редиректов
```

### Шаг 3: Проверьте редирект трейлинг слеша

Возможно, происходит редирект с `/` на `/` или наоборот.

### Шаг 4: Проверка в браузере

Откройте в браузере `https://mn-ka.ru` и проверьте:
- Открывается ли сайт
- Есть ли бесконечный редирект (зацикливание)
- Какой URL в адресной строке

---

## Диагностика

```bash
# Проверьте полный ответ
curl -v https://mn-ka.ru 2>&1 | grep -i "location\|301\|302"

# Проверьте, что возвращает сайт
curl -L https://mn-ka.ru 2>&1 | head -50
```
