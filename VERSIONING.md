# Система версионирования проекта

## Текущая версия: 1.0.0

## Как это работает

Проект использует автоматическое версионирование на основе файла `VERSION.txt`.

### Формат версии

Версия следует формату **Semantic Versioning (SemVer)**: `MAJOR.MINOR.PATCH`

- **MAJOR** — увеличивается при несовместимых изменениях API
- **MINOR** — увеличивается при добавлении функциональности с обратной совместимостью
- **PATCH** — увеличивается при исправлении ошибок с обратной совместимостью

### Автоматическое обновление версии

При выполнении команды `php artisan deploy`:

1. Система проверяет наличие изменений в проекте
2. Если есть изменения, автоматически увеличивается **PATCH** версия (1.0.0 → 1.0.1)
3. Новая версия сохраняется в `VERSION.txt`
4. Версия включается в сообщение коммита: `Deploy v1.0.1: 2025-12-19 11:01:22`

### Ручное управление версией

Если нужно изменить версию вручную:

1. Отредактируйте файл `VERSION.txt`
2. Укажите нужную версию (например, `2.0.0` для major обновления)
3. Закоммитьте изменения

### Просмотр текущей версии

```bash
cat VERSION.txt
```

Или через команду:

```bash
php artisan --version
```

### Примеры

- **Начальная версия**: `1.0.0`
- **Исправление бага**: `1.0.0` → `1.0.1`
- **Новая функция**: `1.0.5` → `1.1.0` (нужно изменить вручную)
- **Большое обновление**: `1.5.0` → `2.0.0` (нужно изменить вручную)

### Важно

- Версия автоматически увеличивается только при наличии изменений в проекте
- При каждом деплое PATCH версия увеличивается на 1
- Для изменения MAJOR или MINOR версии нужно редактировать `VERSION.txt` вручную
